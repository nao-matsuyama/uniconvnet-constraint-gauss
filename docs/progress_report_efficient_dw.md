# 進捗報告 — 骨シンチ セグメンテーションにおける「効率的・適応的な大受容野」機構

対象: 研究室メンバー全員(この研究の前提知識が無くても読めるように書いています)
最終更新: 2026-07-08

---

## 0. 3行サマリ

- 骨シンチ13クラス seg の大カーネル depthwise を「効率化/受容野を学習可能」にする機構を4種類設計・実装し、**厳密な評価基盤(worst 群 × 境界指標 × 統計検定 × multi-seed)**で検証した。
- 結論: **骨シンチは"局所タスク"なので、受容野(RF)を賢くしても精度は上がらない**(全機構が精度中立)。ただし **separable(rank-1)は精度を落とさず約 +11% 速い**という"タダ飯"の効率化になった。
- 最大の教訓: **worst 群の単一実験で p<1e-9 が出ても、seed を変えるだけの訓練ノイズのことがある。multi-seed 対照が必須**。→ RF が本当に効くタスク(網膜血管 FIVES)へ横展開中。

---

## 1. 背景 — 何をやっているか

**タスク**: 骨シンチ画像の13クラス セマンティックセグメンテーション(頭蓋・胸骨・肋骨・脊椎…)。
**モデル**: UniConvNet-T(大カーネル ConvNet)をバックボーンにした U-Net。
その中核 **ConvMod(RFA)** は 7×7 / 9×9 / 11×11 の **depthwise 大カーネル**で広い受容野(RF)を作る。

**研究の問い**: この大カーネル depthwise を、
1. **もっと効率的に**(パラメータ/計算量を減らして)、
2. **受容野を画像内容に応じて学習可能に**、
しつつ **精度を落とさない(むしろ上げる)** ことはできるか?

「受容野を賢くすれば難しい症例(worst)が良くなるはず」という直感が出発点。

---

## 2. 設計した4つの機構

いずれも ConvMod の a1/a2/a3 の depthwise **だけ**を差し替える(他は不変、事前学習重みも転移)。
`--dw-mode` フラグ1つで切り替えられる。

| 機構 | 中身 | 狙い |
|---|---|---|
| **dilated**(既存) | 学習可能な連続 dilation | RF を疎サンプリングで広げる |
| **A: separable** | K×K を **R本の 1×K・K×1 の和**に分解(rank-R) | 計算量 K²→2RK。ガウス的カーネルなら受容野の形を保つ |
| **B: spectral** | 周波数領域のガウス低域通過 + 動的な帯域切り出し | σ(=RFの広さ)を **per-channel 学習可能**に。FFT で RF の大小に依らず一定コスト |
| **C: spectral_mix** | 周波数包絡を **K個のガウスの混合**に | 異なるスケール(=異なる構造の大きさ)の重ね合わせを表現 |

全機構、**受容野の形が理論上ガウス(=AGD, 集約ガウス分布)**になるよう設計しており、
「dilated のように疎サンプリングで受容野が壊れる」問題を避けている。

---

## 3. 評価の作法(この研究の一番の貢献)

**平均 Dice は機構の差にほとんど反応しない**。実際、全機構で平均 Dice はほぼ 0.939〜0.941 で横並び。
そこで次の"物差し"を整備した:

1. **worst 群解析**: 全 val (955枚) を難易度順に並べ、**worst 10% / 25%** の症例だけで比較。
   臨床的に効くのは「一番苦手な症例が良くなるか」なので。
2. **境界指標**: Dice(領域の重なり)に加え **hd95 / assd(境界距離)/ nsd(境界の一致率)**。
   骨シンチでは臓器の境界がずれると困るので、境界の質を直接測る。
3. **統計検定**: worst 群で baseline とペア比較(Wilcoxon)。
4. **★ multi-seed 対照**: **同じ設定を seed を変えて 2〜3 回**学習し、
   「baseline を seed 変えただけでどれくらいスコアが揺れるか(ノイズ帯)」を測る。
   **効果がこのノイズ帯を超えて初めて"本物"**と認める。

この4番目が今回の決定打になった(§5)。

---

## 4. 主な結果

### 4.1 全機構、精度は baseline と中立(骨シンチは局所タスク)

平均も worst 群も、どの機構も baseline と**統計的に区別できない**。
→ **骨シンチは各画素が近傍情報だけで解ける"局所タス"**なので、受容野を広げても・学習可能に
しても精度は動かない。これは negative だが**明確な結論**(RF機構の価値はこのタスクには無い)。

### 4.2 機構ごとの"性格"(効率・受容野・境界)

| 機構 | 実速度(img/s) | params / FLOPs | 精度 | 備考 |
|---|---|---|---|---|
| baseline(dense) | 99 | 36.54M / 19.58G | 基準 | — |
| **A: separable rank-1** | **110 (+11%)** | **35.86M / 18.42G** | 中立 | **唯一、実速度も削減** |
| A: separable rank-2 | 101 | 36.03M / 18.55G | 中立 | rank上げても利得なし |
| B: spectral(学習σ) | 60〜76 (遅い) | 36.56M | 中立 | FFTオーバーヘッドで壁時計は負け。σは実際に学習・per-ch分化する |
| C: spectral_mix | 58 | 36.61M | 中立 | 受容野の集約ガウス性はネイティブ保持 |

- **separable rank-1 が実務的な勝ち**: 精度そのまま、**約11%高速・パラメータ/FLOPs最小**。
- **spectral 系は FFT のオーバーヘッドで実速度は負け**(理論FLOPsは低いのに壁時計は悪化)。
- σ(受容野の広さ)は専用の高い学習率を与えれば**確かに学習し、チャネルごとに分化**する
  (=「各層・各チャネルで受容野を変える」設計目標は機構として実現)。ただし骨シンチでは精度に効かない。

### 4.3 受容野(ERF)の質

separable と dense は受容野が滑らかなガウス(AGD保持)。spectral は pure 版で受容野が尖り気味に崩れるが、
局所枝を併用すれば回復する。→ 機構は「設計どおり受容野を制御できている」ことを定量確認。

---

## 5. 一番の教訓 — 「worst の p<1e-9」でも信じてはいけない

途中、**単一実験**では機構Aが baseline より worst 群を改善するように見えた
(precision worst10% で **p=8×10⁻¹⁰** など、強い有意)。しかし **seed を 3 本に増やす**と:

| worst10% の指標 | baseline の seed 揺れ | 機構A(3seed) | 判定 |
|---|---|---|---|
| dice | +0.0011 〜 +0.0027 | +0.0023 〜 +0.0037 | **揺れの中=ノイズ** |
| precision | −0.0016 〜 **+0.0042** | +0.0005 〜 +0.0071 | **揺れの中=ノイズ** |
| assd(境界) | +0.007 〜 **+0.53** | −0.010 〜 +0.020 | **揺れの中=ノイズ** |

**baseline を seed 変えただけで、precision は 0.006・境界 assd は 0.53 も揺れる**。機構Aの"改善"は
この揺れの中に完全に収まっていた。**つまり単一実験の有意(p<1e-9 すら)は run-to-run の訓練ノイズだった**。

この教訓は過去にも同研究で繰り返し出ている(以前は「正則化が worst を改善」も対照実験でノイズと判明)。
**→ 提言: worst 群での主張は必ず multi-seed でノイズ帯を出してから。単一実験の Wilcoxon 有意は当てにならない。**

---

## 6. 結論と今後

**確定した主張**:
1. 骨シンチは局所タスク。**効率的・適応的な大受容野は精度に効かない**(全機構中立、multi-seed で確定)。
2. **separable rank-1 は精度中立のまま約 +11% 高速・省パラメータ**=タダ飯の効率化。
3. **評価基盤(worst×境界×multi-seed)自体が貢献**。「平均Diceで横並びでも、単一実験の worst 有意は
   ノイズ」を実証。

**今後の2本柱**:
- **(並行)FIVES 網膜血管セグメンテーションへ横展開**: 細い血管は長距離の連続性が要る=**受容野が
  本当に効くタスク**。骨シンチで「機構は正しく動くが局所ゆえ中立」と固めたので、FIVES では
  「RFが効くタスクで初めて精度に効くか」を主眼に検証する(別セッションで進行中)。
- **(本セッション)骨シンチ側のモデル改善**: 受容野以外の軸(損失関数・worst症例・クラス不均衡
  =細く小さい胸骨などの弱いクラス)で worst と境界を底上げできるかを検討する。

## 付録: 用語

- **受容野(RF)/ ERF**: 出力1画素が入力のどれだけ広い範囲を見ているか。ERF は実効受容野(実測)。
- **AGD(集約ガウス分布)**: 受容野が滑らかなガウス形になっていること。崩れると受容野に偽の複製が出る。
- **depthwise conv**: チャネルごとに独立にかける畳み込み(パラメータが軽い)。
- **worst 群**: 予測が難しく低スコアな下位10%/25%の症例。臨床ロバスト性の指標。
- **hd95 / assd / nsd**: 境界の距離・一致率(小さい/大きいほど良い)。領域(Dice)では見えない差を捉える。
- **multi-seed**: 乱数シードを変えて複数回学習し、訓練由来のばらつきを測ること。
