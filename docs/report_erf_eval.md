## 1. 概要と結論

学習可能 dilation による受容野(RF)拡大を **ERF 正則化**（rf_spread を目標へ寄せる損失、係数 λ）で誘導し、骨シンチセグメンテーションの精度・ロバスト性への効果を検証した。

**結論（否定的）:**

1. **平均 Dice はどの λ でも不変**（0.941）。RF 拡大は平均精度を上げも下げもしない。
2. **最悪例(worst 群)の Dice に見えた「有意な改善」(+0.002 前後)は、訓練の乱数(シード)ばらつきの範囲内**であり、ERF 拡大の効果とは言えない。以下の2つの対照実験で確定した。
   - **`freezedil` 対照**（dilation を 1 に固定＝RF 拡大ゼロ）でも worst10% Δ=+0.0019, p=0.002 と、処理あり(`erf0.1`, +0.0026)と同等の「有意改善」が出た。
   - **シード帯対照**（同一設定 λ=0 を seed 42/1/7 で3本学習）でも、baseline 同士が worst10% で Δ=+0.0027(p=0.0016), +0.0024(p=0.0010) とばらつき、処理効果と同サイズだった。
3. → 骨シンチでは **RF 拡大が本質的に効く余地が小さい**（タスクが局所情報で解けてしまう）。

**今後のロードマップ:** RF が本質的に効くかを切り分けるため、
- (c) **プローブ課題**（長距離依存の合成タスク）で「RF が効く設定」を用意 → 検証済み（後述の別報告）。
- (a) **コンテンツ適応機構 `ContentAdaptiveDW`**（画素ごとに RF を可変割当）の有効性検証 → 進行中。
- (b) 実データ（TotalSegmentator 等、広域文脈が要る課題）への展開。

## 2. Wilcoxon 検定によるスコア評価（全 6 通りの帰無仮説）

### 2.1 帰無仮説の定義

worst 群（baseline の Dice が低い下位 k%）の各サンプルについて、
2 モデルのペア差 `Δ = (モデルB の Dice) − (モデルA の Dice)` を取り、

- **帰無仮説 H₀:** Δ の母**中央値 = 0**（その worst 群で A と B に系統的な差がない）
- **対立仮説 H₁:** 中央値 ≠ 0（両側）
- 検定: Wilcoxon 符号順位検定（対応のある非パラメトリック検定）

を全ペアで行う。モデル 3 種 × worst 群 2 種で、総当りは **3 ペア × 2 群 = 6 通り**。

### 2.2 全 6 通りの結果

worst 群は baseline の worst10%(n=96) / worst25%(n=239) で固定。

### 2.3 p 値の意味と「有意 ≠ 効果」の罠

p 値は「H₀（差がない）が正しいと仮定したとき、観測されたペア差以上のものが偶然生じる確率」。p<0.05 で H₀ を棄却＝「この 2 モデル間には系統的な差がある」。

**しかし、この検定は『この特定の 2 本の学習済みモデル』の間の差しか見ておらず、『学習をやり直したときの run 間ばらつき』を帰無仮説に含んでいない。** そのため、**処理差ゼロのモデル同士でも p<0.01 で「有意」を出しうる**。実際に本実験の2対照がそれを示した:

| 対照 | 群 | Δmean | p 値 | 意味 |
|------|----|-------|------|------|
| baseline `s1` vs `s42`（同設定・seed 差のみ） | worst10% | +0.0027 | 0.0016 ★★ | 無処理でも「有意」|
| baseline `s7` vs `s42`（同設定・seed 差のみ） | worst10% | +0.0024 | 0.0010 ★★ | 無処理でも「有意」|
| `freezedil`(RF拡大ゼロ) vs baseline | worst10% | +0.0019 | 0.002 ★★ | RF 拡大なしでも「有意」|

処理 `erf0.1` の効果(+0.0026)は、これら**無処理のばらつき(+0.0019〜+0.0027)と同サイズ**。
→ **Wilcoxon の棄却は「2 モデルが違う」の必要条件にすぎず、「処理が原因」の十分条件ではない。** worst-Dice の評価では、複数シードの分散帯を併記しなければ誤った結論を導く。

## 3. ERF の広がり（σ）の解析と目的の明確化

### 3.1 σ とは何か / 何のために見たか

- **σ の定義:** その層の **Effective Receptive Field (ERF) の空間的広がり（標準偏差, 入力画素単位）**。Luo+2017 の方法で「出力中心画素が入力のどの範囲に依存するか」を勾配マップとして測り、その広がりを σ で表す。**σ が大きい = 受容野が広い。**
- **目的:** 「ERF 正則化 λ を強めると、実際に受容野が広がるのか」という **因果の検証**（狙い: λ↑ → σ↑）。

### 3.2 「減少」は何が減っているのか

下表は **包絡線ガウスフィットによる σ（σ_fit）**。λ を上げるほど σ が**減少 = 受容野がむしろ狭くなっている**。狙いの「λ↑ → σ↑」と逆で、因果が成立していない兆候。

| layer | baseline | erf0.01 | erf0.1 | 傾向 |
|-------|----------|---------|--------|------|
| encoder_stage3 | 35.20 | 25.75 | 16.15 | λ で単調に減少 |
| encoder_stage0 | 10.11 | 9.02 | 8.59 | わずかに減少 |
| decoder_up0 | 11.39 | 10.65 | 8.73 | 減少 |

### 3.3 σ_fit の不安定性と σ_moment による再解釈（重要）

**σ_fit（包絡線へのガウスフィット）は、dilated conv 由来の格子状(gridding)な ERF に対して不安定**で、同一モデルの同一層でも測り直すと **stage3 で 29.8 ↔ 16.1** と大きく揺れた。**単一スカラー σ_fit で判断するのは危険。**

**頑健な 2 次モーメント σ（σ_moment, n=300）で測り直すと、上表とは異なる姿が見えた:**

- **中段（encoder_stage1/2）は λ で +26〜27% 拡大**していた（RF は中段で確かに広がっていた）。
- **ボトルネック（encoder_stage3）は収縮**。これは、H/32=16px しかない特徴マップに対し、dilation を上げた大カーネルが**マップ外（パディング）にはみ出し**、はみ出したタップが無効化されるため（論文も「11×11 は stage3 の 14×14 に収まるよう設計」と明記）。
- σ_moment 自体も **シード間でばらつく**（stage3 で 72 vs 88 等）。→ σ 比較も複数シードが要る。

**設計上の含意:** 単一の dilated conv で depthwise を置換すると、サンプリングが疎になり、その層の ERF のガウス性(AGD)が崩れやすい。→ 後述の方針転換（dense depthwise は保持し、`ContentAdaptiveDW` の多スケール混合で AGD を保ちつつ可変 RF を得る）につながる。

## 4. 可視化結果に基づく直接判断のプロセス

**σ・Δ・p などのスカラー要約は補助**とし、**判断主体は以下の図を人が直接見て**行う（指導教官のご指摘②）。生成物は `visualize_erf.py` / `visualize_receptive_field.py` / `visualize_predictions.py`。

| 図 | 何を読み取るか | 判断基準 |
|----|--------------|---------|
| **ERF ヒートマップ（線形/log）** | 受容野の形・広がり・偏り | 中心対称で滑らかに減衰していればガウス的。格子状・偏りは AGD 崩れ |
| **ERF 断面 + ガウス包絡線** | 断面プロファイルがガウスに乗るか | 実測(赤)が包絡ガウス(青)から芯・裾で外れる度合い |
| **2D FFT + 断面** | 周波数領域での AGD（論文の主張は本質的に周波数性質） | 低周波集中で滑らか＝広く滑らかな RF。高周波リップル＝格子状 |
| **推論オーバーレイ（worst 例）** | どの部位でどう失敗/改善したか | baseline と処理で**同じ場所で失敗しているか**（＝実質差なし）を目視 |

**worst 例の判断手順（推奨）:**
1. `visualize_predictions.py --save-all` で baseline と erf0.1 の `all/panels/000〜` を生成。
2. 分割 seed 固定なので**両者の worst 例は同一画像**。左右に並べ、GT/予測/オーバーレイを比較。
3. 「同じ部位が同じように欠けている」なら、スカラーの Δ が有意でも**実質的な差はない**と結論できる。
